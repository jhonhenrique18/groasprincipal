## Directory Purposes

**Root:**
- Purpose: Project root containing Flask app and configuration
- Key files: `app.py` (main application), `config.py` (settings), `models.py` (database schema)

**static/:**
- Purpose: Static web assets served directly without processing
- Contains: CSS, JavaScript, images, uploaded user content
- Key files: `static/css/style.css` (main styling), `static/js/app.js` (client interactions)

**static/css/:**
- Purpose: Stylesheet definitions for both public and admin interfaces
- Contains: `style.css` (public site - 1000+ lines minified), `admin.css` (admin panel styling)

**static/js/:**
- Purpose: Client-side JavaScript for interactivity without full page reload
- Contains: `app.js` with navbar scroll effects, product modal AJAX, GA4 event tracking

**static/img/:**
- Purpose: Logo variations, hero images, social sharing images
- Key files: `logo-verde.png` (primary logo), `og-image-1200x630.png` (social preview)

**static/uploads/:**
- Purpose: User-uploaded product images via admin panel
- Generated: Yes (created at runtime)
- Committed: No (images generated by admin actions)
- Naming: UUID hex + original extension (e.g., `09e9628098da474cb500c0d384eb35e5.png`)

**templates/:**
- Purpose: Jinja2 templates rendered server-side to HTML
- Contains: Page templates extending `base.html`, admin templates extending `admin/base.html`
- Key files: `base.html` (shared layout for public site), `admin/base.html` (admin layout)

**templates/admin/:**
- Purpose: Admin panel interface templates
- Contains: Login, dashboard, CRUD forms for products/categories, settings management
- Pattern: Single-responsibility forms; `category_form.html` and `product_form.html` handle both create and edit

**.planning/codebase/:**
- Purpose: Generated documentation directory (part of GSD workflow)
- Generated: Yes (created by GSD mapper)
- Committed: Yes (documents tracked in git)
- Contains: `ARCHITECTURE.md`, `STRUCTURE.md`, `CONVENTIONS.md`, etc.

## Naming Conventions

**Files:**

- Python files: lowercase with underscores (`app.py`, `models.py`, `config.py`)
- CSS files: lowercase with hyphens (`style.css`, `admin.css`)
- JavaScript files: lowercase with hyphens (`app.js`)
- Template files: lowercase with hyphens (`base.html`, `product_form.html`)
- Uploaded images: lowercase UUID hex + extension (e.g., `09e9628098da474cb500c0d384eb35e5.png`)

**Directories:**

- Python modules: lowercase, direct names (`models`, `templates`, `static`)
- Subdirectories: feature-based organization (`templates/admin`, `static/css`, `static/uploads`)

**Route URLs:**

- Public routes: lowercase, plural nouns for listings (e.g., `/productos`, `/api/producto/<slug>`)
- Admin routes: prefixed with `/admin`, descriptive paths (e.g., `/admin/categorias`, `/admin/producto/nuevo`)
- REST patterns: RESTful conventions for CRUD (GET for read, POST for create/update, POST for delete via form)

**Variables:**

- Python: snake_case throughout (`product_id`, `category_name`, `admin_logged_in`)
- HTML/Jinja2: snake_case for context variables (`featured`, `current_cat`, `total_products`)
- JavaScript: camelCase for client-side (e.g., `openProductModal()`, `closeModal()`)
- CSS classes: kebab-case (e.g., `product-card`, `modal-title`, `admin-sidebar`)

## Where to Add New Code

**New Public Feature (e.g., product search):**

1. **Backend Route:** Add route in `app.py` after line 235 (after public routes section)
   - Query database: `Product.query.filter_by(...).all()`
   - Return: `render_template('search.html', results=results)`

2. **Template:** Create `templates/search.html` extending `base.html`
   - Use same navbar/footer as other pages (via base.html inheritance)
   - Add SEO blocks for title/meta/og tags in template head

3. **JavaScript (if needed):** Add event listeners in `static/js/app.js`
   - Follow existing patterns: DOM query, event listener, AJAX fetch if dynamic data needed

4. **Styling:** Add CSS rules to `static/css/style.css`
   - Use CSS variables defined in `:root` (colors, spacing, shadows)

**New Admin Feature (e.g., bulk product upload):**

1. **Backend Route:** Add route in `/admin` section of `app.py` (after line 405)
   - Use `@login_required` decorator
   - Handle GET (show form) and POST (process upload)

2. **Template:** Create `templates/admin/bulk_upload.html` extending `templates/admin/base.html`
   - Navigation will auto-integrate via admin sidebar in base.html
   - Follow existing form patterns: form validation, flash messages

3. **Model Changes (if needed):** Update `models.py` if new database fields required
   - Add SQLAlchemy columns
   - Run `db.create_all()` at startup or manual migration

**New Database Model:**

1. Add class to `models.py` inheriting from `db.Model` with:
   - `__tablename__` attribute
   - `db.Column` definitions for fields
   - Optional `to_dict()` method if API endpoint needed

2. Update `seed.py` if model should be populated with initial data
   - Add loop in `seed()` function to create instances from seed_data.json

**Utilities:**

- **Shared helpers:** Add functions to top of `app.py` in helpers section (after line 17)
  - Slug generation: use existing `slugify()`
  - File operations: use existing `save_image()`
  - Authentication: use existing `login_required` decorator

- **New filters/context processors:** Add after line 74 in `app.py`
  - Pattern: `@app.context_processor` decorator
  - Return dict of variables available in all templates

## Route Structure Overview

**Public Routes:**

```
GET  /                    → index.html (homepage)
GET  /productos           → productos.html (all products)
GET  /productos/<slug>    → productos.html (category filtered)
GET  /producto/<slug>     → producto.html (single product detail)
GET  /api/producto/<slug> → JSON (AJAX for modal)
GET  /nosotros            → nosotros.html (about)
GET  /contacto            → contacto.html (GET form, POST submission)
GET  /robots.txt          → robots.txt (SEO)
GET  /sitemap.xml         → sitemap.xml (SEO)
GET  /sitemap-products.xml → sitemap-products.xml (SEO with images)
GET  /uploads/<filename>  → serve uploaded image (Railway volume or static/)
GET  /favicon.ico         → favicon
GET  /404                 → 404.html (error handler)
```

**Admin Routes (all require @login_required):**

```
GET  /admin/login                      → login.html
POST /admin/login                      → session set, redirect dashboard
GET  /admin/logout                     → session clear, redirect login
GET  /admin                            → dashboard.html (stats)
GET  /admin/productos                  → products.html (list)
GET  /admin/producto/nuevo             → product_form.html (create form)
GET  /admin/producto/<id>/editar       → product_form.html (edit form)
POST /admin/producto/nuevo             → process create, redirect list
POST /admin/producto/<id>/editar       → process update, redirect list
POST /admin/producto/<id>/eliminar     → delete, redirect list
GET  /admin/categorias                 → categories.html (list)
GET  /admin/categoria/nueva            → category_form.html (create)
GET  /admin/categoria/<id>/editar      → category_form.html (edit)
POST /admin/categoria/nueva            → process create, redirect list
POST /admin/categoria/<id>/editar      → process update, redirect list
POST /admin/categoria/<id>/eliminar    → delete, redirect list
GET  /admin/configuracion              → settings.html (form)
POST /admin/configuracion              → update WhatsApp/email/hero, redirect
```
